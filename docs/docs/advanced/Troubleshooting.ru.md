# Устранение неполадок

Ниже приведён список распространённых проблем, с которыми вы можете столкнуться при настройке клиента или сервера, а также предлагаемые решения.

## failed to initialize client (connect error: timeout: no recent network activity)

Эта ошибка возникает, когда клиент не может подключиться к серверу. Наиболее частые причины:

- Сервер не запущен
- Порт заблокирован фаерволом (помимо системного, многие провайдеры имеют отдельные настройки фаервола в панели управления)
- Сервер работает на другом адресе или порту
- Сервер слушает на сети, недоступной для клиента
- Доменное имя не разрешается в правильный IP-адрес
- Неправильные настройки обфускации
- Слишком старая версия ядра Linux (известная проблема на CentOS 7). Подробнее см.
  [[1]](https://github.com/apernet/hysteria/issues/810#issuecomment-1807690164)
  [[2]](https://blog.yywq.me/article/39f87fd6-1c3e-4826-90d9-5d031393ca8d)

## failed to initialize client (authentication error, HTTP status code: 404)

Эта ошибка возникает, когда сервер отклоняет клиента. Наиболее частые причины:

- Неправильные учётные данные
- Вы подключились к неправильному серверу
- На сервере неправильно настроена аутентификация

## failed to initialize client (connect error: CRYPTO_ERROR 0x12a (local): tls: failed to verify certificate: x509: certificate signed by unknown authority)

Эта ошибка возникает, когда клиент считает сертификат сервера недействительным. Наиболее частые причины:

- Сервер использует самоподписанный сертификат, и вы не добавили его в доверенные CA клиента и не использовали опцию `insecure`.
- В хранилище доверенных CA вашей системы отсутствует CA, подписавший сертификат сервера.
- Вы подвергаетесь MITM-атаке ([атака «человек посередине»](https://ru.wikipedia.org/wiki/Атака_посредника)) со стороны третьего лица.

## failed to load server config (invalid config: listen: listen udp :443: bind: permission denied)

Эта ошибка возникает, когда у сервера нет разрешения на привязку к указанному порту. Вы можете сделать одно из следующего:

- Запустить сервер от имени root
- Предоставить исполняемому файлу возможность `cap_net_bind_service`: `sudo setcap cap_net_bind_service=+ep ./hysteria-linux-amd64-avx`
